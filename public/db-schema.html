Relations entre tables
users 1 → N posts Un utilisateur peut avoir plusieurs publications
posts 1 → N post_images Un post peut avoir jusqu'à 5 photos
posts 1 → N comments Un post peut avoir plusieurs commentaires
comments 1 → N comments Un commentaire peut avoir des réponses (auto-référence via parent_id)
users N ↔ N comments Via comment_likes — un utilisateur peut liker plusieurs commentaires
users N ↔ N posts Via favorites — un utilisateur peut sauvegarder plusieurs posts
users 1 → N reports Un utilisateur peut signaler plusieurs publications
users (admin) 1 → N notifications Les admins envoient des notifications aux utilisateurs
